# ChoreCop — Telegram Mini App Plan

Move all admin setup (family creation, member management, rule management) into a Telegram Mini App (Web App). The group bot becomes notification + duty-interaction only. Admin work happens in the browser inside Telegram.

---

## Architecture after this change

```
Bot (Telegraf)
  /start, /menu  → sends "Open Settings" Mini App button
  duty handlers  → mark done, request approval, approve/reject (unchanged)
  scheduler      → generate duties, send summaries, send reminders (unchanged)

Web server (Express)
  GET  /            → serves Mini App HTML
  GET  /api/me      → family + member for authenticated user
  POST /api/family  → create family
  GET  /api/members → list members
  POST /api/members → add member
  DEL  /api/members/:id → deactivate member
  GET  /api/rules   → list rules
  POST /api/rules   → create rule
  DEL  /api/rules/:id   → deactivate rule

Mini App (vanilla HTML/JS)
  Screen 1: Setup   — create family (shown when no family exists)
  Screen 2: Members — list, add, remove
  Screen 3: Rules   — list, add (wizard), remove
```

**Authentication:** Every API request sends `X-Telegram-Init-Data` header.
Server validates it via HMAC-SHA256 with the bot token, extracts `user.id`,
looks up the family from that user's linked member record.

## Task 2: Auth middleware — initData validation

**File:** `src/web/auth.ts`

```ts
import crypto from 'crypto';

export interface TelegramUser {
  id: number;
  first_name: string;
  username?: string;
}

export function validateInitData(initData: string, botToken: string): TelegramUser | null {
  const params = new URLSearchParams(initData);
  const hash = params.get('hash');
  if (!hash) return null;
  params.delete('hash');

  const dataCheckString = [...params.entries()]
    .sort(([a], [b]) => a.localeCompare(b))
    .map(([k, v]) => `${k}=${v}`)
    .join('\n');

  const secretKey = crypto.createHmac('sha256', 'WebAppData').update(botToken).digest();
  const expected = crypto.createHmac('sha256', secretKey).update(dataCheckString).digest('hex');

  if (hash !== expected) return null;

  const userStr = params.get('user');
  if (!userStr) return null;
  return JSON.parse(userStr) as TelegramUser;
}
```

**Express middleware** (inline in server.ts):
```ts
function requireAuth(req, res, next) {
  const initData = req.headers['x-telegram-init-data'] as string;
  const user = validateInitData(initData ?? '', process.env.BOT_TOKEN!);
  if (!user) return res.status(401).json({ error: 'Unauthorized' });
  req.telegramUser = user;
  next();
}
```

**`src/web/auth.ts`** also exports a helper for routes to resolve family from `req.telegramUser.id`:
```ts
export function getFamilyForUser(db, telegramId: number): { family: Family; member: Member } | null
```
This queries members by telegram_id, joins to family. Returns null if unlinked.

## Task 3: Express server

**File:** `src/web/server.ts`

- Creates Express app
- `express.json()` middleware
- Serves `src/web/public/` as static files
- Mounts `/api/me`, `/api/family`, `/api/members`, `/api/rules` routes
- Exports `startWebServer(db, port)` function called from `src/index.ts`

```ts
export function startWebServer(db: Database.Database, port: number): void {
  const app = express();
  app.use(express.json());
  app.use(express.static(new URL('./public', import.meta.url).pathname));
  app.use('/api/me', meRouter(db));
  app.use('/api/family', familyRouter(db));
  app.use('/api/members', membersRouter(db));
  app.use('/api/rules', rulesRouter(db));
  app.listen(port, () => console.log(`Web server on :${port}`));
}
```

## Task 4: API routes

### `GET /api/me`
Returns the current user's family and member info.
- If no family linked to user → `{ family: null, member: null }`
- If found → `{ family: Family, member: Member }`

### `POST /api/family`  body: `{ name: string }`
- Creates family with `chat_id` from the Telegram group (needs to be sent from client: `Telegram.WebApp.initDataUnsafe.chat?.id` or `start_param` encoding the chat_id)
- Adds the authenticated user as dad
- Returns `{ family, member }`

> **Note on chat_id:** The Mini App is opened from the group chat, so `Telegram.WebApp.initDataUnsafe.chat.id` contains the group chat ID. The client sends it in the POST body.

### `GET /api/members`
- Returns all active members for the user's family
- 401 if not authenticated, 404 if no family

### `POST /api/members`  body: `{ name: string, role: 'mom' | 'kid' }`
- Adds a member to the authenticated user's family
- Only allowed if `req.member.role === 'dad'`
- Returns the new member

### `DELETE /api/members/:id`
- Sets `active = 0` on the member
- Only dad can do this

### `GET /api/rules`
- Returns all active rules for the family

### `POST /api/rules`
```json
{
  "name": "Посудомийна",
  "schedule": "daily",
  "workers_count": 1,
  "rotation_mode": "round_robin",
  "fixed_member_ids": []
}
```
- Creates rule, sets fixed assignments if mode = "fixed"
- Only dad or mom

### `DELETE /api/rules/:id`
- Sets `active = 0` on the rule
